# 降本绩效管理（数据报表模块）页面设计

**页面名称**：降本绩效管理  
**建议路由**：`/cost-reduction-performance`  
**默认展示**：仅展示“降本”（本月加权均价 < 上月加权均价）的记录  

---

## 1. 目标与背景

采购降本页面核心是：基于采购员的采购记录，以 **采购员 + 供应商 + SKU** 的维度进行月度汇总，对比“本月 vs 上月”的采购加权均价，筛选出存在降价（降本）的数据，并输出降本单价、降本比例、降本金额等指标，用于绩效管理与月度复盘。

---

## 2. 统计口径（强约束）

### 2.1 统计维度（分组键）

以以下字段作为分组键（Group Key）聚合：
- `purchaser`（采购员）
- `supplier`（供应商）
- `sku`（库存 SKU）

同一个 `sku` 的展示信息（缩略图/名称）由 SKU 映射得到（示例数据阶段可直接冗余在聚合结果中）。

### 2.2 月度汇总（本月/上月一致）

对某个选定月份 `M`（例如 2026-04）：
- 本月采购数量 `qty_cur` = Σ(采购数量)
- 本月采购金额 `amt_cur` = Σ(采购数量 * 采购单价)
- 本月加权均价 `price_cur` = `amt_cur / qty_cur`（qty_cur=0 时为 null）

对上月 `M-1`：
- 上月采购数量 `qty_prev`、上月采购金额 `amt_prev`、上月加权均价 `price_prev` 同理计算

### 2.3 降本计算（已确认选项）

仅当 `price_prev` 与 `price_cur` 均存在且 `price_cur < price_prev` 时，该分组记为“降本记录”，进入列表展示：

- **降本单价** `save_unit` = `price_prev - price_cur`
- **降本比例** `save_rate` = `(price_prev - price_cur) / price_prev`（price_prev=0 时显示 `--`）
- **降本金额** `save_amt` = `max(0, price_prev - price_cur) * qty_cur`
  - 因为仅展示降本记录，理论上 `save_unit > 0`，但仍用 `max(0, …)` 防守

---

## 3. 页面结构（UI/UX）

页面布局采用“顶部筛选区 + 下方表格”的报表页结构（参考现有 `/purchaser-performance`、`/supplier-kpi` 列表页模式）。

### 3.1 筛选区（查询）

字段与交互：
- **采购员**：下拉单选，支持关键字模糊查询（默认：全部）
- **供应商**：下拉单选，支持关键字模糊查询（默认：全部）
- **库存SKU/名称**：输入框关键字模糊匹配
- **日期筛选**：按月（单选月份，例如 YYYY-MM）

按钮：
- 搜索：按当前筛选条件刷新列表
- 重置：恢复默认筛选值（采购员=全部、供应商=全部、SKU/名称空、月份=当前月）
- 导出：按当前月份 + 当前筛选条件导出

### 3.2 表格字段（列定义）

列（从左到右建议顺序，可按密度调整）：
1. 采购员
2. 供应商
3. 商品（SKU/名称，双行展示：上行 SKU、下行 名称）
4. 缩略图
5. 本次采购单价（本月加权均价，保留 4 位小数）
6. 上次采购单价（上月加权均价，保留 4 位小数）
7. 降本数据（同列双行展示）
   - 第 1 行：降本单价（元/件，保留 4 位小数）
   - 第 2 行：降本比例（百分比，保留 2 位小数；上月均价=0 显示 `--`）
8. 本次采购数量（本月数量汇总）
9. 降本金额（元，保留 2 位小数）

默认排序建议：
- 降本金额倒序（更符合“绩效贡献”优先查看）
- 次级排序：降本单价倒序

---

## 4. 数据与示例（实现阶段用 mock）

### 4.1 推荐的数据形态（供实现阶段落地）

**输入原始采购记录（mock source）**：
- purchaser
- supplier
- sku
- productName
- thumbnailUrl
- month（YYYY-MM）
- qty
- unitPrice

**输出聚合行（table row）**：
- purchaser, supplier, sku, productName, thumbnailUrl
- qty_cur, price_cur
- qty_prev, price_prev
- save_unit, save_rate, save_amt

### 4.2 过滤规则

列表只展示：
- `price_prev != null && price_cur != null && price_cur < price_prev`

筛选条件会作用于聚合结果（而不是原始记录），例如：
- 采购员筛选匹配 `purchaser`
- 供应商筛选匹配 `supplier`
- SKU/名称关键字匹配 `sku` 或 `productName`

---

## 5. 导出（按月）

### 5.1 导出范围

导出范围与列表一致：
- 当前选择月份 `M`
- 当前筛选条件
- 仅导出降本记录（与列表保持一致）

### 5.2 导出内容

导出字段与表格列一致（推荐导出为 CSV；如果后续需要 Excel 可再扩展）。

### 5.3 导出文件命名建议

`降本绩效管理_YYYY-MM.csv`（可追加筛选摘要，但避免过长）

---

## 7. 边界与异常处理

- 上月无数据：`price_prev = null`，该记录不展示（因为无法判定降本）
- 本月无数据：不展示
- 上月均价=0：`save_rate` 显示 `--`，`save_unit` 可正常计算（通常为 0-本月均价，若本月均价也为 0 则不展示）
