# 添加商品（/products/add）功能设计

日期：2026-05-06

## 背景与目标

补齐「添加商品」页面的表单能力，覆盖基础信息、扩展属性、仓库信息三块，并实现：
- 表单校验（必填、SKU 特殊字符禁用、SKU 唯一性、数值范围）
- 商品目录/品牌的“下拉选择 + 管理弹框（增删改）”，本地持久化
- 仓库信息的联动计算（实时计算但允许人工覆盖）
- 保存后写入本地数据源并返回商品列表，且新商品可见

本期不对接后端接口，使用 localStorage 持久化。

## 入口与路由

- 商品列表：`/products`
- 添加商品：`/products/add`（现为占位页，需要实现完整表单）

## 数据源（localStorage）

### 商品数据
- Key：`products_data_v1`
- 写入时机：
  - `products` 页面首次加载：若 key 不存在，用当前 mock 初始化并写入
  - `products/add` 保存成功：追加写入 key，并跳转回 `/products`

### 商品目录
- Key：`product_categories_v1`
- 结构：支持二级目录（ERP 主分类 / 商品类目）

### 品牌
- Key：`product_brands_v1`

### 供应商起订量（数据来源说明）
- “生产商起订量”不在添加商品页录入；来源于采购模块/供应商管理在库存 SKU 关联供应商时设置。
- 添加商品页第一版在“未关联供应商（无起订量）”时默认按 0 处理，最小采购量按候选值计算并允许人工覆盖。

## 页面信息架构

### 顶部操作区
- 标题：添加商品
- 按钮：
  - 保存并返回（主按钮）
  - 取消（返回 `/products`，不保存）

### 表单分区
1. 基本信息
2. 扩展属性
3. 仓库信息（自建本地仓）

## 字段设计与校验规则

### A. 基本信息

#### 必填字段

1) 库存SKU（必填）
- 组件：文本输入框
- 去空格：保存前对首尾空格 trim
- 校验：
  - 非空
  - 不允许包含以下任意字符（黑名单）：`’、“、!、$、?、_、{、}`
  - 唯一性：`products_data_v1` 内 sku 不可重复（大小写敏感，按原值判重）
- 错误文案：
  - “库存SKU不能为空”
  - “库存SKU不支持特殊字符：’、“、!、$、?、_、{、}”
  - “库存SKU已存在，请更换”

2) 中文名称（必填）
- 组件：文本输入框
- 校验：非空
- 错误文案：“中文名称不能为空”

3) 状态（必选）
- 组件：下拉单选
- 枚举：
  - 自动创建
  - 等待开发
  - 正常销售
  - 商品清仓
  - 停止销售

4) 商品形态（必选）
- 组件：下拉单选
- 枚举：实物商品 / 虚拟商品

#### 非必填字段（第一版不强校验）
- 主SKU：文本
- 英文名称：文本
- 原厂SKU：文本
- 商品目录：下拉单选（来自 `product_categories_v1`），旁边“目录管理”
- 品牌：下拉单选（来自 `product_brands_v1`），旁边“品牌管理”
- 序列号管控、目录报关编码、申报品名英文/中文、申报价格、单品重量、报关规格型号、原产地、境内货源地、征免：文本/数字输入（按你现有页面结构先给输入框占位，不做深度校验）
- 单位：下拉单选（示例枚举：个/支/套/箱/件）
- 产品销售链接：文本
- 开发员：下拉单选（示例枚举：张伟/李娜/王强/刘美希/陈刚，可后续从组织架构接入）
- 库存图片：图片在线地址（文本）
- 虚拟SKU：文本
- 商品备注 / 销售备注 / 采购备注：文本

### B. 扩展属性（全部非必填）

组件：文本输入框（按字段逐项输入）
- 品牌类型、材质、是否背心、类别、成分含量、是否有领、是否有扣、是否有拉链

第一版不做枚举强校验。

### C. 仓库信息（自建本地仓）

#### 必填字段

1) 仓库（必选）
- 组件：下拉单选
- 示例枚举：东莞周转仓 / 义乌周转仓 / 深圳周转仓（可按你实际清单替换）

#### 业务字段（数字默认 0，允许为空时按 0 计算）

- 库存数量（数字）
- 盘点周期（数字，单位：天）
- 盘点员（下拉单选，示例枚举同“开发员”）
- 采购天数（数字）
- 库存警戒天数（数字）
- 库存警戒（数字）
- 预测日销量(个)（数字，第一版新增输入框，默认 0）
- 最小采购量（数字）
- 采购上限（数字，范围 0~100000）

#### 联动计算（实时计算 + 可编辑覆盖）

联动字段：
- 库存警戒（默认值）
- 最小采购量（默认值）

参与计算字段：
- 预测日销量
- 采购天数
- 库存警戒天数

规则：
1) 库存警戒默认值
- `库存警戒 = 库存警戒天数 * 预测日销量`

2) 最小采购量默认值
- 候选值：`候选值 = 预测日销量 * 采购天数`
- 起订量：来自“供应商管理-库存SKU关联供应商”设置（若当前 SKU 暂无关联供应商或无起订量，按 0 处理）
- `最小采购量 = max(候选值, 起订量)`

覆盖策略：
- 用户手动修改“库存警戒/最小采购量”后，将该字段标记为“手动覆盖”，后续不再自动覆盖该字段
- 提供“恢复默认计算值”按钮（每个联动字段各一个或统一按钮均可）：点击后取消手动覆盖标记，并立即按公式刷新默认值

#### 数值范围校验
- 采购上限：必须在 0~100000（含边界）
- 其它数字字段：本期不做上限强校验，但禁止 NaN/非数字输入（输入为空视为 0）

## 商品目录管理弹框（下拉 + 管理）

入口：基本信息“商品目录”字段右侧按钮“目录管理”

展示字段：
- 目录名称
- 描述
- 报关编码
- 多属性（是/否）
- 层级：ERP 主分类 / 商品类目（通过父子关系表达）

能力：
- 新增：可选父级（为空则为一级目录）；填写名称等字段
- 编辑：修改名称/描述/报关编码/多属性
- 删除：
  - 若该目录被任意商品引用（products_data_v1 内 categoryId/categoryPath），提示“目录已被商品引用，无法删除”
  - 否则允许删除（删除二级目录时不影响父级）

持久化：保存到 `product_categories_v1`，添加页下拉立即可用。

## 品牌管理弹框（下拉 + 管理）

入口：基本信息“品牌”字段右侧按钮“品牌管理”

能力：
- 新增/编辑/删除品牌名称
- 删除限制：若品牌被商品引用（products_data_v1 内 brandId），提示“品牌已被商品引用，无法删除”

持久化：保存到 `product_brands_v1`，添加页下拉立即可用。

## 保存与反馈

保存按钮：保存并返回
- 行为：
  1) 执行全表单校验（必填、SKU 黑名单、SKU 唯一性、采购上限范围）
  2) 校验失败：
     - 页面顶部提示“请修正 X 处错误”
     - 自动滚动并聚焦到第一处错误
  3) 校验成功：
     - 写入 `products_data_v1`（新商品插入到数组头部，便于返回后立即看到）
     - toast 提示“保存成功”
     - 跳转回 `/products`

取消按钮：
- 直接跳转回 `/products`，不写入数据

## 列表可见性（/products）

- `/products` 需要从 `products_data_v1` 渲染数据（而非组件内写死 mock）
- 保存返回后，新商品在列表顶部出现

## 验收标准（关键）

- 库存SKU：必填、黑名单字符校验、唯一性校验生效
- 中文名称/状态/商品形态：必填校验生效
- 目录/品牌：可在添加页打开管理弹框增删改，且返回下拉立即可选
- 仓库信息：
  - 仓库必选
  - 预测日销量/采购天数/库存警戒天数输入后，库存警戒自动计算并可覆盖
  - 最小采购量默认按 `max(预测日销量*采购天数, 起订量)`；未关联供应商时起订量按 0
  - “恢复默认计算值”可恢复自动计算
  - 采购上限范围校验生效
- 保存成功：返回商品列表，新商品可见
