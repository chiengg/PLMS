## 背景

单品SKU列表中「7/28/42销量」可点击打开「日销量/剩余库存趋势」弹框。现有弹框支持：

- 折线图：日销量、剩余库存
- 图例点击切换显示/隐藏（至少保留一条）
- 示例数据：基于 SKU 等信息生成稳定序列

本需求在弹框中新增「去年销量同比」折线图及对应图例，并支持图例控制显示/隐藏。

## 目标

- 新增折线：去年同期“日销量同比”
- 新增图例项：总和-去年同期日销量
- 默认展示三条线：总和-日销量、总和-剩余库存、总和-去年同期日销量
- 通过点击图例控制展示/隐藏，且至少保留一条折线

## 非目标

- 不接入真实后端销量数据（仍为示例数据）
- 不新增导出功能（本次仅同比折线与图例）
- 不调整弹框其它筛选项交互与布局（除新增图例项）

## 交互与视觉

- 图例区新增第三项「总和-去年同期日销量」
- 三个图例均可点击切换显示/隐藏
- 如果用户试图隐藏最后一条可见折线，操作无效

颜色规范：

- 总和-日销量：蓝色（沿用现有）
- 总和-剩余库存：粉色（沿用现有）
- 总和-去年同期日销量：绿色（新增）

## 数据口径（示例数据）

为保证同一 SKU 在同一筛选条件下曲线稳定一致，示例数据采用“稳定种子 + 伪随机序列”方式生成。

- 今年日销量序列 seed：`{sku}|{warehouse}|{platform}|daily`
- 剩余库存序列 seed：`{sku}|{warehouse}|{platform}|stock`
- 去年同期日销量序列 seed：`{sku}|{warehouse}|{platform}|daily_yoy`

X 轴展示当前筛选区间的日期（MM-DD），同比折线按序号对齐展示，不额外切换为去年日期标签。

## 代码变更范围

- SalesTrendDialog：新增第三条折线数据、图例项与显示/隐藏状态管理，更新图表绘制逻辑（多折线）
- utils：沿用现有 buildSalesSeries 生成序列

## 验收标准

- 打开弹框默认显示三条折线，颜色区分符合规范
- 点击任意图例可隐藏/显示对应折线
- 不允许全部隐藏导致空图（至少保留一条可见折线）
- 通过类型检查与现有测试集（`npm run check`、`npm run test:run`）
